## 1. 接口总览

包含 16+ 个 API 端点的完整列表:
  - 认证接口 (4个)
  - 训练管理接口 (4个)
  - 兼容性检查接口 (4个)
  - 数据集管理接口 (3个)

每个端点都标注了:
  - HTTP 方法
  - 是否需要认证
  - 简要说明

## 2. 认证接口

详细说明 4 个认证相关接口:
  
  POST /api/register
    - 请求体: username, email, password
    - 响应: user_id
    - 字段验证规则
  
  POST /api/login
    - 请求体: username, password
    - 响应: user 对象
    - Session Cookie 设置
  
  POST /api/logout
    - 无请求体
    - 清除 Session
  
  GET /api/check_auth
    - 检查当前认证状态
    - 返回用户信息

## 3. 训练管理接口

详细说明 4 个训练相关接口:
  
  POST /api/start_training
    - 请求体: 完整的训练配置
    - 响应: session_id
    - 参数详细说明（通用 + 问题特有 + 策略特有 + 算法特有）
  
  GET /api/training_progress/<session_id> (SSE)
    - Server-Sent Events 实时推送
    - 7种消息类型
    - 连接管理
  
  GET /api/training_status/<session_id>
    - 获取当前状态快照
    - 返回: status, progress, epoch, loss, reward
  
  POST /api/stop_training/<session_id>
    - 停止训练任务

## 4. 兼容性检查接口

详细说明 4 个兼容性相关接口:
  
  GET /api/compatibility/info
    - 获取完整兼容性矩阵
  
  GET /api/compatibility/constraints/<problem>
    - 获取问题支持的策略和算法
  
  POST /api/compatibility/validate
    - 验证配置组合是否有效
    - 返回警告信息
  
  GET /api/compatibility/recommend
    - 获取推荐配置
    - 支持 3 种偏好: best, fast, simple

## 5. SSE 实时消息

详细说明 7 种消息类型:
  
  1. info - 信息消息
     示例: "开始训练 POMO 模型"
  
  2. warning - 警告消息
     示例: "无法从trainer获取loss和reward"
  
  3. error - 错误消息
     示例: "训练出错: ..."
  
  4. progress - 训练进度
     包含: epoch, total_epochs, progress, loss, reward, best_reward, plot_url
  
  5. plot - 图表更新
     包含: plot_url, message
  
  6. complete - 训练完成
     包含: message, results (完整的训练结果)
  
  7. heartbeat - 心跳消息
     保持连接活跃（30秒间隔）

在编写文档过程中，总结了几个关键规范:

### 规范 1: 返回字段名必须一致

前端期望的字段名:
  - plot_paths (静态对比图)
  - animation_paths (动画)
  - training_curve (训练曲线)
  - checkpoint_path (检查点)

后端必须使用相同的字段名:
  ✅ return {'plot_paths': [...]}
  ❌ return {'comparison_paths': [...]}  # 错误

### 规范 2: 路径类型区分

两种路径:
  
  本地文件路径（用于文件操作）:
    /full/path/to/static/model_plots/user_1/file.png
  
  URL 路径（返回给前端）:
    /static/model_plots/user_1/file.png

使用场景:
  - 生成文件时 → 本地路径
  - 保存记录时 → 本地路径
  - 返回前端时 → URL 路径

### 规范 3: SSE 消息类型

7 种标准消息类型:
  - info: 信息
  - warning: 警告
  - error: 错误（会结束连接）
  - progress: 进度更新
  - plot: 图表更新
  - complete: 完成（会结束连接）
  - heartbeat: 心跳

### 规范 4: 训练配置参数

4 类参数:
  1. 通用参数: problem, model, algorithm, num_loc, epochs, batch_size, learning_rate
  2. 问题特有参数: num_agents, cost_type, vehicle_capacity, time_window_size
  3. 策略特有参数: num_starts, embed_dim, num_encoder_layers, num_heads
  4. 算法特有参数: clip_ratio, value_loss_coef, entropy_coef

### 规范 5: BaseTrainer 属性

子类可用的属性:
  ✅ self.user_plots_dir (图表目录)
  ✅ self.user_checkpoints_dir (检查点目录)
  ✅ self.bg_file_manager (文件管理器)
  ✅ self.bg_session_manager (会话管理器)
  ❌ self.plots_dir (不存在)
  ❌ self.checkpoints_dir (不存在)
