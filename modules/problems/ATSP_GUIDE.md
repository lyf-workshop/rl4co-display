# ATSP (非对称旅行商问题) 使用指南

## 📋 问题简介

### 什么是ATSP？

**ATSP (Asymmetric Traveling Salesman Problem，非对称旅行商问题)** 是经典TSP的扩展版本，核心区别在于：

- **TSP**: 距离矩阵对称，即 `d(A,B) = d(B,A)`
- **ATSP**: 距离矩阵不对称，即 `d(A,B) ≠ d(B,A)`

### 实际应用场景

1. **城市交通路线**
   - 考虑单行道、交通管制
   - 上下坡、风向影响

2. **航空路线规划**
   - 考虑风向、气流
   - 航线高度限制

3. **网络路由**
   - 单向链路
   - 不对称带宽

4. **物流配送**
   - 考虑道路等级（高速→省道容易，反之难）
   - 城区限行规则

## 🎯 问题特征

| 特征 | 说明 |
|------|------|
| **距离矩阵** | 非对称 (n×n矩阵，不满足 d[i][j] = d[j][i]) |
| **路径约束** | 每个城市访问一次，返回起点 |
| **容量约束** | 无 |
| **时间约束** | 无 |
| **难度** | 比TSP更高（搜索空间更大） |
| **NP-hard** | 是 |

## 🔧 使用方法

### 1. 在Web界面中使用

#### 步骤1：选择问题类型

在训练页面的"问题配置"区域，选择：
```

```

#### 步骤2：配置参数

```
🏙️ 问题规模: 50  (建议范围: 10-200)
```

**参数说明**：
- `num_loc`: 城市数量
  - 最小值: 5（体现非对称性）
  - 推荐值: 20-100
  - 最大值: 1000（计算复杂度过高）

#### 步骤3：选择模型

**推荐配置**：
- ✅ **最佳性能**: Attention Model + PPO
- ⚡ **快速训练**: Attention Model + A2C
- 🚫 **不支持**: POMO（POMO设计用于对称问题）

**为什么POMO不支持ATSP？**
- POMO利用问题对称性进行多起点并行采样
- ATSP的非对称性使得POMO的优化策略失效

#### 步骤4：开始训练

点击"开始训练"按钮，系统会自动：
1. 生成非对称距离矩阵
2. 启动训练
3. 实时显示进度
4. 生成可视化结果

## 📊 与TSP的对比

| 维度 | TSP | ATSP |
|------|-----|------|
| **距离矩阵** | 对称 (n×n/2 个独立距离) | 非对称 (n×n 个独立距离) |
| **复杂度** | O(n!) | O(n!) |
| **搜索空间** | 较小 | 更大 |
| **可用策略** | AM, POMO | 仅 AM（POMO不支持） |
| **训练难度** | 中等 | 较高 |
| **收敛速度** | 较快 | 较慢 |

## 🎨 可视化特点

ATSP的可视化会显示：
1. **路线图**：城市位置和访问顺序
2. **方向性**：箭头表示路径方向（重要！）
3. **距离标注**：显示非对称距离
4. **对比图**：训练前后的路径对比

**注意**：由于ATSP的非对称性，可视化会特别强调路径的方向性。

## ⚠️ 注意事项

### 1. 策略限制

- ✅ **支持**: Attention Model (AM)
- 🚫 **不支持**: POMO（会自动阻止或警告）
- 原因：POMO的多起点策略依赖对称性

### 2. 训练建议

- **算法选择**：
  - 推荐 PPO 或 A2C（比REINFORCE更稳定）
  - 避免使用 REINFORCE（收敛困难）

- **规模建议**：
  - 测试: 10-20个城市
  - 标准: 50-100个城市
  - 大规模: 100-200个城市（需GPU）

- **训练轮数**：
  - ATSP通常需要比TSP更多的训练轮数（+20%-50%）

### 3. 性能预期

ATSP比TSP更难训练，预期表现：
- 收敛速度：比TSP慢20%-40%
- 最终性能：取决于非对称程度
- GPU加速：强烈推荐（提升10-30倍）

### 距离矩阵生成

RL4CO的ATSPEnv会自动生成非对称距离矩阵：
- 基于随机坐标
- 添加非对称性扰动
- 确保三角不等式满足

### 奖励计算

ATSP的奖励为负的总路径长度：
```
reward = -total_distance
```

其中总距离考虑方向性：
```
total_distance = Σ distance_matrix[i][j]  (按访问顺序)
```

## 🐛 常见问题

### Q1: 为什么选不了POMO？

**答**：POMO专为对称问题设计，不支持ATSP。系统会自动禁用或提示警告。请使用Attention Model。

### Q2: ATSP训练比TSP慢很多？

**答**：正常现象。ATSP的搜索空间更大，建议：
- 使用PPO或A2C算法（比REINFORCE稳定）
- 适当增加训练轮数
- 使用GPU加速

### Q3: 如何验证距离矩阵的非对称性？

**答**：可以在训练日志中查看：
- 系统会打印距离矩阵的对称性检查
- 可视化图中会显示方向箭头

### Q4: ATSP的最优解如何评估？

**答**：
- 小规模(n<20): 可与精确算法对比
- 中大规模: 与启发式算法(如OR-Tools)对比
- 评估改进百分比（相比随机策略）
