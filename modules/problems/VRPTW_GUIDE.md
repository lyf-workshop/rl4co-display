# VRPTW 问题类型说明文档

## 📋 概述

**VRPTW (Vehicle Routing Problem with Time Windows)** - 带时间窗的车辆路径问题

这是VRP问题的一个重要扩展，除了容量约束外，还增加了**时间窗约束**：每个客户必须在指定的时间段内被访问。

---

## 🎯 问题特点

### 基本约束

1. **容量约束**
   - 每辆车有固定的载货容量
   - 访问客户会消耗容量
   - 容量不足时必须返回仓库补货

2. **时间窗约束** ⭐核心特性
   - 每个客户有一个时间窗口 `[earliest, latest]`
   - 必须在时间窗内开始服务
   - 如果到达过早，需要等待
   - 如果到达过晚（硬时间窗），则不可行

3. **服务时间**
   - 每个客户需要一定的服务时间
   - 服务时间计入总时间

4. **旅行时间**
   - 从一个地点到另一个地点需要旅行时间
   - 旅行时间 = 距离 / 速度

## 📊 时间窗类型

### 硬时间窗 (Hard Time Windows)

- **定义**：必须严格遵守时间窗约束
- **违反后果**：解决方案不可行（拒绝）
- **适用场景**：
  - 医疗配送（必须在指定时间送达）
  - 预约上门服务（客户只在特定时间在家）
  - 学校班车（必须准时接送）

### 软时间窗 (Soft Time Windows)

- **定义**：可以违反时间窗，但会产生惩罚成本
- **违反后果**：解决方案可行，但成本增加
- **适用场景**：
  - 普通快递（可以稍晚送达）
  - 外卖配送（延迟会扣分但不拒单）
  - 一般物流配送

## 🔍 技术细节

### 问题复杂度

- **VRPTW是NP-hard问题**
- **比CVRP更复杂**（多了时间维度）
- **约束更严格**（容量 + 时间窗）

### 复杂度对比

| 问题类型 | 约束数量 | 决策维度 | 相对难度 |
|---------|---------|---------|---------|
| TSP | 0 | 1维（空间） | ★☆☆☆☆ |
| CVRP | 1 | 1维（空间） | ★★★☆☆ |
| SDVRP | 1 | 1维（空间） | ★★★☆☆ |
| **VRPTW** | 2 | 2维（空间+时间） | ★★★★★ |

### 解决策略

1. **时间窗约束处理**
   - 早到：等待到最早开始时间
   - 晚到：软时间窗记录惩罚，硬时间窗拒绝

2. **路线优化目标**
   - 最小化总距离
   - 最小化总时间
   - 最小化时间窗违反惩罚

3. **强化学习方法**
   - 策略网络：Attention Model（推荐）
   - 算法：PPO或A2C（更稳定）
   - 奖励函数：负成本（距离+惩罚）

**特点**：

- 必须准时送达
- 硬时间窗

- 必须在预约时间内到达
- 硬时间窗

## ⚠️ 注意事项

### 1. 问题规模限制

```
建议客户数 ≤ 100
原因：时间窗约束使问题复杂度大幅增加
```

### 2. 参数设置合理性

```python
# ❌ 不合理
time_window_width = 500  # 时间窗太宽（失去约束意义）
max_time = 100           # 最大时间太短（无法完成）

# ✅ 合理
time_window_width = 100  # 适中的时间窗
max_time = 480           # 8小时工作时间
```

### 3. 算法选择

```
❌ 不推荐：REINFORCE（训练不稳定）
✅ 推荐：PPO或A2C（更稳定）
```

### 4. 时间窗类型选择

```
硬时间窗：要求严格，适合预约服务
软时间窗：允许违反，适合一般配送
```
