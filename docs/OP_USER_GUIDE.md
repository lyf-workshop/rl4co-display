# OP（定向问题）使用指南

## 📋 问题简介

**OP (Orienteering Problem，定向问题)** 是一个经典的路径优化问题，源自定向越野运动，广泛应用于旅游规划、无人机巡检、物流配送等领域。

### 问题定义

- **地点（Locations）**：每个地点有一个奖励值（prize/score）
- **Depot**：起点和终点（必须返回）
- **路径长度约束**：总路径长度不能超过 `max_length`
- **选择性访问**：不需要访问所有地点
- **配对关系**：无配对约束（与 PDP 不同）

### 优化目标

在满足路径长度约束的前提下，**最大化收集的奖励总和**

```
maximize: Σ prize(visited_locations)
subject to: total_path_length ≤ max_length
```

## 🎯 使用场景

OP 适用于以下优化场景：

### 1. **旅游路线规划** 🗺️
- **场景**：有限时间内游览尽可能多的高评分景点
- **奖励**：景点评分/热度
- **约束**：游览时间/行程距离
- **示例**：一日游规划、城市观光路线

### 2. **无人机巡检** 🚁
- **场景**：电池限制下巡检关键区域
- **奖励**：区域重要性/风险等级
- **约束**：电池续航距离
- **示例**：电力线路巡检、森林火情监测

### 3. **移动传感器数据采集** 📡
- **场景**：有限移动预算下采集高价值数据
- **奖励**：数据价值/信息熵
- **约束**：移动成本/电量
- **示例**：环境监测、信号采样

### 4. **应急救援** 🚑
- **场景**：有限时间内救助尽可能多的高危伤员
- **奖励**：伤员危险等级
- **约束**：救援时间窗口
- **示例**：灾害救援、战地医疗

### 5. **营销拜访规划** 💼
- **场景**：销售人员有限时间内拜访高价值客户
- **奖励**：客户潜在价值
- **约束**：工作时间/交通时间
- **示例**：B2B销售、客户维护

## 🔄 OP vs 其他问题

| 特性 | OP（定向）| TSP（旅行商）| CVRP（车辆路径）| PDP（取送货）|
|------|----------|------------|----------------|-------------|
| 访问所有点 | ❌ 选择性 | ✅ 必须 | ✅ 必须 | ✅ 必须 |
| 路径长度约束 | ✅ 有 | ❌ 无 | ❌ 无 | ❌ 无 |
| 奖励/价值 | ✅ 最大化 | ❌ 无 | ❌ 无 | ❌ 无 |
| 容量约束 | ❌ 无 | ❌ 无 | ✅ 有 | 可选 |
| 配对约束 | ❌ 无 | ❌ 无 | ❌ 无 | ✅ 有 |
| 优化目标 | **最大化奖励** | 最小化距离 | 最小化距离 | 最小化距离 |

**OP 的核心特点**：
1. **选择性访问** - 这是与 TSP 最大的区别
2. **奖励导向** - 优化目标是最大化而非最小化
3. **路径约束** - 增加了决策难度

## ⚙️ 参数配置

### 基础参数

| 参数 | 说明 | 推荐范围 | 默认值 |
|------|------|----------|--------|
| **地点数量（num_loc）** | 可选择访问的地点数（不含depot）| 10-100 | 20 |
| **最大路径长度（max_length）** | 路径长度上限 | 1.0-5.0 | 2.0 |
| **奖励类型（prize_type）** | 奖励值的生成方式 | dist/unif/const | dist |

### 参数详解

#### 1. 地点数量（num_loc）

- **含义**：可供选择访问的地点总数（不包括 depot）
- **影响**：
  - 地点少（10-20）：问题简单，训练快
  - 地点多（50-100）：选择空间大，更具挑战性
- **推荐**：
  - 快速测试：20
  - 标准配置：50
  - 挑战性：100

#### 2. 最大路径长度（max_length）

- **含义**：从 depot 出发并返回的总路径长度上限
- **预定义值**：
  - 20 地点 → 2.0
  - 50 地点 → 3.0
  - 100 地点 → 4.0
- **调整建议**：
  - 更宽松（+0.5）：可以访问更多地点
  - 更严格（-0.5）：增加难度，需要更精细选择

#### 3. 奖励类型（prize_type）

##### dist - 距离奖励（推荐）⭐
- **定义**：奖励 = 地点距离 depot 的距离
- **特点**：距离越远，奖励越高
- **适用场景**：鼓励探索远处的高价值区域
- **示例**：远郊景点价值更高

##### unif - 均匀随机
- **定义**：奖励在 [min_prize, max_prize] 范围内均匀随机分布
- **特点**：奖励分布均匀
- **适用场景**：模拟随机的客户价值
- **示例**：客户价值随机分布

##### const - 常量奖励
- **定义**：所有地点奖励值相同
- **特点**：问题简化为在约束下访问最多地点
- **适用场景**：地点价值相同，只关注覆盖率
- **示例**：传单派发、数据采集点

### 推荐配置

#### 小规模问题（快速测试）
```
地点数量：20
最大路径长度：2.0
奖励类型：dist（距离奖励）
训练轮数：100-200
```

#### 中等规模问题（标准配置）
```

最大路径长度：3.0
奖励类型：dist（距离奖励）
训练轮数：500-1000
```

#### 大规模问题（挑战性）
```

奖励类型：unif（均匀随机）
训练轮数：1000-2000
```

## 🚀 训练流程

### 1. 选择问题类型

在主界面的"问题类型"下拉菜单中选择：
```
OP - 定向问题 ⭐新增
```

### 2. 配置定向问题参数

展开"定向问题参数"区域，设置：
- **地点数量**：20（推荐，快速训练）
- **最大路径长度**：2.0（根据地点数量自动建议）
- **奖励类型**：dist - 距离奖励（推荐）

💡 **提示**：首次使用建议选择默认配置！

### 3. 选择策略和算法

**推荐组合：**
- **策略**：Attention Model（AM）
- **算法**：PPO（最佳质量）或 A2C（快速训练）

> ⚠️ 注意：OP 不支持 POMO 策略

### 4. 设置训练参数

```
批次大小：128-256（推荐 256）
学习率：0.0001
训练轮数：500-1000
```

### 5. 启动训练

点击"开始训练"按钮，系统将：
1. 验证参数合法性
2. 初始化 OP 环境（生成奖励值）
3. 开始强化学习训练
4. 实时推送训练进度

## 💡 优化技巧

### 1. 参数调优

**地点数量影响：**
- 地点少（10-20）：选择空间小，容易训练
- 地点多（50-100）：选择空间大，需要更强的策略

**路径长度影响：**
- 宽松（+20%）：可以访问更多地点，容易获得高奖励
- 严格（-20%）：约束更强，需要精细选择高价值地点

**奖励类型影响：**
- `dist`：鼓励探索远处，路线更分散
- `unif`：随机分布，需要学习识别高价值区域
- `const`：简化问题，转为最大覆盖问题

### 2. 训练策略

**学习率选择：**
- 标准：0.0001
- 收敛慢：0.0005
- 不稳定：0.00005

**批次大小：**
- 小批次（64-128）：更新频繁
- 大批次（256-512）：训练稳定

**训练轮数：**
- 小规模：100-500 轮
- 中等规模：500-1000 轮
- 大规模：1000-2000 轮

### 3. 常见问题排查

**问题：收集奖励太低**
- 增加训练轮数
- 尝试 PPO 算法（更稳定）
- 适当增加 max_length

**问题：访问地点太少**
- 检查 max_length 是否过于严格
- 增加训练轮数
- 使用 dist 奖励类型（鼓励探索）

**问题：路径超出限制**
- 模型训练不足（增加轮数）
- 检查环境参数是否正确
- 验证 max_length 设置

**问题：可视化显示异常**
- 检查后端日志
- 验证数据生成是否正确
- 确认前端接收到数据

## 🛠️ 技术细节

### 环境定义

使用 RL4CO 库的 `OPEnv`：
```python
from rl4co.envs.routing import OPEnv

env = OPEnv(
    generator_params={
        'num_loc': 20,
        'prize_type': 'dist',
        'max_length': 2.0
    }
)
```

### 状态表示

- **地点坐标**：depot + 所有可选地点
- **奖励值**：每个地点的 prize
- **已访问标记**：哪些地点已被访问
- **当前位置**：车辆当前所在地点
- **剩余路径长度**：还可以行驶的距离

### 动作空间

每个决策点选择：
- 访问某个未访问的地点（如果路径允许）
- 返回 depot（结束行程）

### 约束条件

1. **路径长度约束**：total_length ≤ max_length
2. **唯一性约束**：每个地点最多访问一次
3. **起终点约束**：必须从 depot 出发并返回 depot

### 奖励设计

```
reward = Σ prize(visited_locations)
```

**特点**：
- OP 是最大化问题（与 TSP/CVRP 的最小化不同）
- 奖励越高越好
- 需要平衡"访问更多地点"和"路径长度限制"

## ❓ 常见问题（FAQ）

### Q1：OP 与 TSP 有何区别？
**A**：
- **TSP**：必须访问所有地点，目标是最小化总路径长度
- **OP**：选择性访问，在路径长度限制内最大化奖励

OP 更灵活，适合资源受限场景。

### Q2：为什么不访问所有地点？
**A**：因为有路径长度约束（max_length）。如果访问所有地点会超出约束，模型会学习选择高价值地点。

### Q3：如何理解"距离奖励"（dist）？
**A**：
- 奖励 = 地点距离 depot 的欧几里得距离
- 距离越远，奖励越高
- 鼓励模型探索远处的地点
- 模拟"远郊景点更有价值"的场景

### Q4：max_length 如何设置？
**A**：
- **预定义值**（推荐）：
  - 20地点 → 2.0
  - 50地点 → 3.0
  - 100地点 → 4.0
- **自定义**：
  - 访问率过低 → 增加 max_length
  - 问题太简单 → 减少 max_length

### Q5：如何判断训练效果好坏？
**A**：
1. **奖励趋势**：训练曲线应上升并趋于稳定
2. **访问率**：在合理范围（20-60%），不是越高越好
3. **路径利用率**：接近但不超过 100%
4. **高价值地点优先**：可视化中应优先访问大点（高奖励）

### Q6：OP 能否用于多车辆？
**A**：当前实现是单车辆 OP。如果需要多车辆：
- 运行多次单车辆 OP
- 或考虑使用 Team Orienteering Problem (TOP)

### Q7：奖励值在可视化中如何体现？
**A**：
- **点的大小**：越大表示奖励越高
- **颜色深度**：黄→橙→红（奖励递增）
- **数值标注**：直接显示奖励值
- **访问状态**：绿色实心表示已访问，灰色空心表示未访问

## 🎓 高级技巧

### 1. 奖励值分布策略

**高价值集中型**：
- 使用 `dist` 奖励
- 较小的 max_length
- 适合"少而精"的场景

**均衡覆盖型**：
- 使用 `const` 奖励
- 较大的 max_length
- 适合"多多益善"的场景

**混合策略型**：
- 使用 `unif` 奖励
- 中等的 max_length
- 适合模拟真实复杂场景

### 2. 路径规划策略

**激进策略**：
- max_length 接近理论最大值
- 追求访问更多地点
- 风险：可能无法返回 depot

**保守策略**：
- max_length 留有余量
- 确保高价值地点访问
- 更稳定可靠
