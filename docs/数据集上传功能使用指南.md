# TSP 数据集上传功能使用指南

## 功能概述

现在系统支持用户上传自己的TSP数据集进行训练和测试。用户可以使用自定义的城市坐标数据，而不是使用系统随机生成的数据。

## 支持的文件格式

### 1. TXT 格式（推荐用于简单数据）

每行一个坐标对，格式：`x y`

```txt
# 这是注释，会被忽略
0.5 0.5
0.2 0.3
0.8 0.7
0.1 0.9
```

### 2. JSON 格式（推荐用于程序生成）

标准JSON格式，包含coordinates数组：

```json
{
  "coordinates": [
    [0.1, 0.1],
    [0.2, 0.3],
    [0.3, 0.2]
  ]
}
```

### 3. TSPLIB 格式（标准格式）

符合TSPLIB规范的TSP文件：

```
NAME: test10
TYPE: TSP
DIMENSION: 10
EDGE_WEIGHT_TYPE: EUC_2D
NODE_COORD_SECTION
1 20 30
2 40 50
3 60 20
EOF
```

## 使用步骤

### 步骤 1: 准备数据集文件

- 创建一个包含城市坐标的文件（TXT、JSON或TSP格式）
- 确保坐标数据格式正确
- 建议城市数量在 10-100 之间

### 步骤 2: 上传数据集

1. 在训练首页找到"数据集上传 (TSP)"卡片
2. 将"数据集模式"切换为"上传自定义数据集"
3. 点击"选择数据集文件"按钮，选择你的文件
4. 点击"📤 上传数据集"按钮
5. 等待上传完成，系统会显示数据集信息（文件名、城市数量、坐标范围）

### 步骤 3: 使用数据集训练

1. 选择模型类型（如 Attention Model）
2. 配置训练参数（轮数、学习率等）
3. 点击"🚀 开始训练"
4. 系统会使用你上传的数据集进行训练和测试

### 步骤 4: 管理数据集

- 点击"📂 管理已上传数据集"查看所有已上传的数据集
- 可以选择已有数据集重新使用
- 可以删除不需要的数据集

## 功能特性

### ✅ 已实现

1. **多格式支持**：支持TXT、JSON、TSPLIB三种格式
2. **数据验证**：自动验证文件格式和数据有效性
3. **数据集管理**：查看、选择、删除已上传的数据集
4. **训练集成**：训练时使用上传的数据集
5. **测试评估**：在上传的数据集上测试模型性能
6. **可视化结果**：生成路径对比图和动态GIF

### 🔄 工作原理

1. **训练阶段**：系统使用随机生成的数据进行模型训练（保证泛化能力）
2. **测试阶段**：使用您上传的数据集进行测试，生成可视化结果
3. **结果展示**：显示在您的数据集上的路径规划结果和成本

## 测试数据集

项目根目录提供了三个测试数据集文件：

1. **test_dataset_10cities.txt** - 10个城市的TXT格式数据
2. **test_dataset_20cities.json** - 20个城市的JSON格式数据
3. **test_dataset_tsplib.tsp** - 15个城市的TSPLIB格式数据

您可以使用这些文件测试上传功能。

## 注意事项

### 📌 重要提示

1. **坐标范围**：建议坐标值在 0-1 之间（归一化），或在合理的数值范围内
2. **城市数量**：
   - 最少：5个城市
   - 推荐：10-50个城市
   - 最多：100个城市（更多可能导致训练时间过长）
3. **文件大小**：单个文件建议不超过1MB
4. **数据隔离**：每个用户的数据集独立存储，互不干扰

### ⚠️ 常见问题

**Q: 上传失败怎么办？**
A: 检查文件格式是否正确，坐标数据是否有效，文件编码是否为UTF-8

**Q: 可以上传真实世界的城市坐标吗？**
A: 可以！但需要先将经纬度坐标归一化到合理范围

**Q: 数据集存储在哪里？**
A: 存储在服务器的 `datasets/user_<用户ID>/` 目录下

**Q: 训练时使用的是上传的数据集吗？**
A: 训练使用随机生成的数据（提高泛化能力），测试使用上传的数据集

## API 接口

### 上传数据集
```
POST /api/upload_dataset
Content-Type: multipart/form-data
Body: dataset=<文件>
```

### 查看数据集列表
```
GET /api/list_datasets
```

### 删除数据集
```
POST /api/delete_dataset
Content-Type: application/json
Body: {"dataset_id": "xxx"}
```
