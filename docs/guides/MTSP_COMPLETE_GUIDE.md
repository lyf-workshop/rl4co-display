## 快速开始

### 什么是 mTSP？

**mTSP (Multiple Traveling Salesman Problem)** = 多个旅行商协同完成所有城市的访问

**核心特点**：
- 🚗 多个车辆/代理 (agents)
- 📍 从同一起点 (depot) 出发
- 🎯 访问所有城市
- 🔄 最后都返回起点
- ⚖️ 目标：均衡负载或最小化总成本

### 3步开始训练

**第 1 步：选择问题类型**
- 在训练页面选择 **mTSP - 多旅行商问题**

**第 2 步：配置参数**
```
城市数量: 50
代理数量: 5
优化目标: minmax（均衡负载）或 sum（最小总成本）
```

**第 3 步：选择训练算法**
- 策略模型: **Attention Model** 或 **POMO**
- 训练算法: REINFORCE / PPO / A2C
- 训练轮次: 10-30

## 问题介绍

### mTSP vs TSP

| 特性 | TSP | mTSP |
|------|-----|------|
| 车辆数 | 1个 | 多个 |
| 路径数 | 1条 | 多条 |
| 优化目标 | 最短路径 | 最大均衡 / 最短总和 |
| 复杂度 | NP-hard | NP-hard（更复杂） |

### 实际应用场景

1. **物流配送**：多辆配送车辆协同配送
2. **巡检任务**：多个无人机巡检区域
3. **机器人调度**：多机器人仓库拣货
4. **服务派遣**：多维修人员上门服务

### 两种优化目标

**minmax（最小化最大值）**：
- 目标：让最忙的车辆尽量轻松
- 适用：均衡负载，避免某辆车过度劳累
- 示例：5辆车的路径长度 [10, 11, 10, 9, 10]

**sum（最小化总和）**：
- 目标：让所有车辆的总成本最小
- 适用：最小化总运营成本
- 示例：5辆车的路径长度 [15, 8, 12, 5, 10]（总和50）

## 常见问题

### Q1: minmax 和 sum 如何选择？

**使用 minmax 当**：
- 需要均衡负载
- 避免某辆车过度使用
- 关注最差情况

**使用 sum 当**：
- 需要最小化总成本
- 关注总体效率
- 不在意负载不均

---

### Q2: 代理数量如何确定？

**经验公式**：
```
代理数 ≈ 城市数 / 10
```

**示例**：
- 20个城市 → 2-3个代理
- 50个城市 → 4-6个代理  
- 100个城市 → 8-12个代理

**注意**：代理太少（路径太长），代理太多（效率低）

---

### Q3: 训练效果不好怎么办？

**优化方案**：

1. **切换优化目标**
   ```
   如果 minmax 效果差，试试 sum
   ```

2. **增加训练轮次**
   ```
   10 → 30 → 50
   ```

3. **切换策略模型**
   ```
   Attention Model → POMO
   ```

4. **使用更强算法**
   ```
   REINFORCE → PPO
   ```

---

### Q4: 如何验证结果正确性？

**验证清单**：

1. ✅ 每个城市都被访问一次
2. ✅ 所有代理都返回起点
3. ✅ 没有重复访问
4. ✅ 训练后成本降低

**代码验证**：
```python
# 检查城市覆盖
visited = set()
for route in routes:
    visited.update(route)
assert len(visited) == num_cities
```

### 警告: 某些代理路径为空

**警告信息**: `Agent 3 has empty route`

**原因**: 代理数量过多，某些代理分配不到城市

**解决**: 减少代理数量或增加城市数量
