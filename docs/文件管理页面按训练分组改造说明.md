# 文件管理页面按训练分组改造说明

## 改造目标

将文件管理页面从**平铺展示**改为**按训练会话分组展示**，每次训练的所有相关文件（对比图、动态GIF、训练曲线、检查点）聚合在一起，便于用户管理和使用。

---

## 主要改动

### 1. 后端API改造 (`app.py`)

#### 1.1 修改 `/api/list_files` 接口

**改动位置**: `app.py` 第 1788-1842 行

**主要变化**:
- **之前**: 返回按文件类型分组的平铺列表
  ```python
  {
    'plots': [...],
    'checkpoints': [...],
    'total_count': 50
  }
  ```

- **现在**: 返回按训练会话分组的结构化数据
  ```python
  {
    'sessions': [
      {
        'session_id': 'abc123',
        'model_type': 'attention',
        'problem_type': 'tsp',
        'status': 'completed',
        'start_time': '2024-01-01 10:00',
        'file_count': 4,
        'total_size_mb': 15.3,
        'files': {
          'comparison': {...},      # 对比图
          'animation': {...},       # 动态GIF
          'curve': {...},          # 训练曲线
          'checkpoint': {...},     # 检查点
          'other': [...]           # 其他文件
        }
      },
      ...
    ],
    'total_sessions': 10,
    'total_files': 40,
    'total_size_mb': 150.5
  }
  ```

**核心逻辑**:
```python
# 1. 获取用户的所有训练会话
user_sessions = session_mgr.get_user_sessions(user_id, limit=100)

# 2. 遍历每个会话，获取其文件
for session in user_sessions:
    # 查询该会话的所有文件
    cursor.execute("""
        SELECT * FROM training_files 
        WHERE session_id = %s AND user_id = %s
    """, (session_id, user_id))
    
    # 3. 按文件类型分类（每种类型只保留第一个）
    files_by_type = {
        'comparison': None,
        'animation': None,
        'curve': None,
        'checkpoint': None
    }
```

#### 1.2 新增 `/api/delete_session` 接口

**位置**: `app.py` 第 2089-2146 行

**功能**: 删除整个训练会话及其所有文件

**核心逻辑**:
```python
@app.route('/api/delete_session', methods=['POST'])
@login_required
def delete_session():
    # 1. 验证会话所有权
    if not session_mgr.verify_session_owner(session_id, user_id):
        return 403
    
    # 2. 获取该会话的所有文件
    cursor.execute("""
        SELECT * FROM training_files 
        WHERE session_id = %s AND user_id = %s
    """, (session_id, user_id))
    
    # 3. 逐个删除文件（包括数据库记录和磁盘文件）
    for file_record in session_files:
        file_manager.delete_file(file_record['id'], user_id)
```

### 2. 前端页面改造 (`templates/file_manager.html`)

#### 2.1 样式调整

**新增样式类**:
- `.sessions-grid` - 训练会话网格容器
- `.session-card` - 单个训练会话卡片
- `.session-header` - 会话头部（显示模型、问题、状态）
- `.session-status` - 状态徽章（已完成/进行中/失败）
- `.file-item` - 文件项（对比图/GIF/曲线/检查点）

**布局变化**:
```
之前:

现在:
```

## 数据流程图

```
用户访问文件管理页面
    ↓
前端: loadFiles()
    ↓
GET /api/list_files
    ↓
后端: list_training_files()
    ├─ 获取用户的训练会话列表
    ├─ 遍历每个会话
    │   └─ 查询该会话的所有文件
    ├─ 按文件类型分类
    └─ 返回分组数据
    ↓
前端: renderSessions()
    ├─ 渲染每个训练会话卡片
    ├─ renderFileItem() 渲染文件项
    └─ 显示查看/下载/删除按钮
    ↓
用户操作
    ├─ 查看文件 → viewFile()
    ├─ 下载文件 → downloadFile() / downloadCheckpoint()
    ├─ 删除文件 → confirmDelete() → POST /api/delete_file
    └─ 删除会话 → confirmDeleteSession() → POST /api/delete_session
```

## 使用示例

### 场景1: 查看最近的训练结果
1. 打开文件管理页面
2. 最新的训练会话显示在最上方
3. 点击"👁️ 查看"按钮查看对比图
4. 点击"👁️ 查看"按钮查看动态GIF

### 场景2: 下载特定训练的所有文件
1. 找到目标训练会话卡片
2. 逐个点击"📥 下载"按钮
3. 或点击"🗑️ 删除整个训练"前先全部下载

### 场景3: 清理旧的训练文件
1. 找到不需要的训练会话
2. 点击"🗑️ 删除整个训练"
3. 确认对话框 → 一次性删除该训练的所有文件

## 总结

本次改造大幅提升了文件管理的**用户体验**和**操作效率**:

| 指标 | 改造前 | 改造后 | 提升 |
|------|-------|-------|------|
| **文件关联识别** | 需手动识别文件名 | 自动按训练分组 | ⭐⭐⭐⭐⭐ |
| **删除效率** | 逐个删除4个文件 | 一键删除整个训练 | **提升4倍** |
| **信息展示** | 仅文件名 | 模型/问题/状态/时间 | ⭐⭐⭐⭐⭐ |
| **页面整洁度** | 平铺列表混乱 | 结构化卡片清晰 | ⭐⭐⭐⭐⭐ |

**核心价值**: 将"文件管理"升级为"训练管理"，用户关注点从单个文件转向整个训练会话，更符合实际使用场景。
