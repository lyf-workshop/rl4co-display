# 自定义数据集功能修复说明

## 问题描述

在使用自定义TSP数据集时，出现错误：
```
'key "done" not found in TensorDict with keys ['locs']'
```

## 问题原因

原代码手动创建的 `TensorDict` 只包含了 `locs`（坐标）字段：

```python
# ❌ 错误的方式
td_init = TensorDict({
    'locs': coords_tensor,
}, batch_size=[1]).to(device)
```

但是 RL4CO 的 TSP 环境需要完整的状态信息，包括：
- `locs`: 城市坐标
- `done`: 是否完成标志
- `action_mask`: 动作掩码（哪些城市可以访问）
- `visited`: 已访问城市的标记
- `current_node`: 当前节点
- 等其他状态字段

## 解决方案

使用环境的 `reset()` 方法创建完整的状态结构，然后替换坐标数据：

```python
# ✅ 正确的方式
# 1. 先用环境创建完整状态（包含所有必要字段）
td_init = env.reset(batch_size=[1]).to(device)

# 2. 然后替换坐标为自定义数据集
coords_tensor = torch.tensor([custom_dataset], dtype=torch.float32).to(device)
td_init['locs'] = coords_tensor
```

## 修复详情

### 修改位置：app.py 第 1160-1176 行

**修改前：**
```python
if custom_dataset is not None:
    coords_tensor = torch.tensor([custom_dataset], dtype=torch.float32).to(device)
    td_init = TensorDict({
        'locs': coords_tensor,
    }, batch_size=[1]).to(device)
```

**修改后：**
```python
if custom_dataset is not None:
    # 首先使用环境的reset创建完整的状态结构（包含所有必要字段）
    td_init = env.reset(batch_size=[1]).to(device)
    
    # 然后替换其中的坐标为自定义数据集
    coords_tensor = torch.tensor([custom_dataset], dtype=torch.float32).to(device)
    td_init['locs'] = coords_tensor
```

## 技术说明

### 为什么这样修复有效？

1. **完整性**：`env.reset()` 创建的状态包含所有 TSP 环境需要的字段
2. **兼容性**：环境自动初始化所有状态字段（done=False, visited=空, 等）
3. **灵活性**：我们只替换坐标数据，其他状态字段保持环境的初始化值
4. **一致性**：确保状态结构与环境期望的格式完全一致

### TSPEnv 的完整状态结构

一个典型的 TSP 状态 TensorDict 包含：

```python
TensorDict({
    'locs': tensor([[...]]),        # 城市坐标 [batch, num_cities, 2]
    'action_mask': tensor([...]),   # 动作掩码 [batch, num_cities]
    'visited': tensor([...]),       # 已访问标记 [batch, num_cities]
    'current_node': tensor([...]),  # 当前节点索引 [batch]
    'i': tensor([...]),             # 当前步数 [batch]
    'done': tensor([...]),          # 是否完成 [batch]
}, batch_size=[1])
```
