# PDP（取送货问题）使用指南

## 📋 问题简介

**PDP (Pickup and Delivery Problem)** 是经典的物流优化问题，广泛应用于快递配送、出租车调度、无人机配送等场景。

### 问题定义

- **取货点（Pickup）**：需要装载货物的地点
- **送货点（Delivery）**：需要卸载货物的地点
- **配对约束**：每个取货点都有对应的送货点，必须先访问取货点，再访问其对应的送货点
- **Depot**：车辆的出发点和终点

### 优化目标

最小化 **总路径长度**：完成所有取送货任务的总行驶距离

## 🎯 使用场景

PDP 适用于以下物流调度场景：

1. **快递配送**
   - 从多个商家取货，配送到多个客户
   - 上门取件，送至配送中心
   - 逆向物流（退货取件）

2. **出租车调度**
   - 乘客上车点（pickup）和下车点（delivery）
   - 多乘客拼车路线规划
   - 动态需求响应

3. **货物运输**
   - 集装箱运输（取箱点 → 还箱点）
   - 零担货运（多点取货，多点送货）
   - 冷链物流配送

4. **无人机配送**
   - 仓库取货，客户送货
   - 医疗物资紧急配送
   - 偏远地区物资投放

## 🔄 PDP vs 其他问题

| 特性 | PDP（取送货）| CVRP（车辆路径）| TSP（旅行商）|
|------|------------|----------------|-------------|
| 配对约束 | ✅ 先取后送 | ❌ 无配对 | ❌ 无配对 |
| 容量约束 | 可选 | ✅ 有容量 | ❌ 无容量 |
| 访问顺序 | 严格（取→送）| 灵活 | 灵活 |
| 应用场景 | **配送调度** | 配送优化 | 路线优化 |
| 复杂度 | 高 | 中等 | 较低 |

**PDP 的核心特点** 是取送货的**配对约束**，这使得问题复杂度显著增加。

## ⚙️ 参数配置

### 基础参数

| 参数 | 说明 | 推荐范围 | 默认值 |
|------|------|----------|--------|
| **地点数量（num_loc）** | 取送货地点总数（不含depot）| 4-40（必须是偶数）| 20 |
| **取送货对数** | = num_loc / 2 | 2-20 | 10 |
| **强制从depot开始** | 是否必须先访问depot | true/false | false |

**重要：** `num_loc` 必须是偶数，因为取货点和送货点成对出现。

### 参数说明

#### 地点数量（num_loc）

- **含义**：取送货地点的总数（不包括depot）
- **计算**：num_loc = 取货点数量 + 送货点数量 = 2 × 取送货对数
- **示例**：
  - num_loc = 20 → 10个取货点 + 10个送货点（10对）
  - num_loc = 10 → 5个取货点 + 5个送货点（5对）

#### 强制从depot开始

- **false（默认）**：车辆可以从任意取货点开始，最后返回depot
  - 更灵活，路径更优
  - 推荐用于实际应用
- **true**：车辆必须先访问depot，然后开始配送
  - 更严格的约束
  - 适用于车辆必须从仓库出发的场景

### 推荐配置

#### 小规模问题（快速测试）
```
地点数量：10（5对）
强制从depot开始：否
训练轮数：100-200
```

#### 中等规模问题（标准配置）
```
地点数量：20（10对）
强制从depot开始：否
训练轮数：500-1000
```

#### 大规模问题（挑战性）
```
地点数量：40（20对）
强制从depot开始：否
训练轮数：1000-2000
```

## 🚀 训练流程

### 1. 选择问题类型

在主界面的"问题类型"下拉菜单中选择：
```
PDP - 取送货问题 ⭐新增
```

### 2. 配置取送货参数

展开"取送货问题参数"区域，设置：
- **地点数量**：20（推荐，10对取送货）
- **强制从depot开始**：否（推荐）

💡 **提示**：地点数量必须是偶数！

### 3. 选择策略和算法

**推荐组合：**
- **策略**：Attention Model（AM）
- **算法**：PPO（最佳质量）或 A2C（快速训练）

> ⚠️ 注意：PDP 不支持 POMO 策略

### 4. 设置训练参数

```
批次大小：128-256（推荐 256）
学习率：0.0001
训练轮数：500-1000
```

### 5. 启动训练

点击"开始训练"按钮，系统将：
1. 验证参数合法性（地点数量是否为偶数）
2. 初始化 PDP 环境
3. 开始强化学习训练
4. 实时推送训练进度

对比模型生成的路线和基准解：

- **左图**：模型解（RL算法生成）

**解读要点：**
- 路线越短越好
- 配对约束必须满足（先取后送）

### 3. 训练曲线

## 💡 优化技巧

### 1. 参数调优

**地点数量影响：**
- 地点少（10-20）：问题简单，训练快
- 地点多（30-40）：约束复杂，需更多训练

**强制depot影响：**
- 不强制（false）：路径更灵活，成本更低
- 强制（true）：约束更严格，符合实际需求

### 2. 训练策略

**学习率选择：**
- 标准：0.0001
- 收敛慢：0.0005
- 不稳定：0.00005

**批次大小：**
- 小批次（64-128）：更新频繁
- 大批次（256-512）：训练稳定

**训练轮数：**
- 小规模：100-500 轮
- 中等规模：500-1000 轮
- 大规模：1000-2000 轮

### 3. 常见问题排查

**问题：路径长度过大**
- 增加训练轮数
- 尝试 PPO 算法（更稳定）
- 检查参数是否合理

**问题：训练不收敛**
- 降低学习率
- 增加批次大小
- 简化问题规模

**问题：可视化显示异常**
- 检查后端日志
- 验证数据生成是否正确
- 确认前端接收到数据

**问题：配对约束违反**
- 检查模型训练是否完成
- 增加训练轮数
- 尝试不同的策略/算法组合

## 🛠️ 技术细节

### 环境定义

使用 RL4CO 库的 `PDPEnv`：
```python
from rl4co.envs.routing import PDPEnv

env = PDPEnv(
    generator_params={
        'num_loc': 20,  # 20个地点 = 10对取送货
    },
    force_start_at_depot=False
)
```

### 状态表示

- **地点坐标**：depot + 取货点 + 送货点
- **已访问标记**：哪些地点已被访问
- **可送货标记**：哪些送货点可以访问（对应取货点已访问）
- **当前位置**：车辆当前所在地点

### 动作空间

每个决策点选择：下一个要访问的地点（受配对约束限制）

### 约束条件

1. **配对约束**：必须先访问取货点 P_i，才能访问送货点 D_i
2. **唯一性约束**：每个地点只能访问一次
3. **完整性约束**：所有地点都必须访问

### 奖励设计

```
reward = -path_length（路径越短，奖励越高）
```

## ❓ 常见问题（FAQ）

### Q1：PDP 与 CVRP 有何区别？
**A**：
- **PDP**：有配对约束，必须先取后送，每个需求有两个地点（取货点和送货点）
- **CVRP**：无配对约束，只有送货点，有车辆容量限制

### Q2：为什么 num_loc 必须是偶数？
**A**：因为 PDP 中每个需求包含一个取货点和一个送货点，它们成对出现。例如 num_loc=20 表示 10 个取货点 + 10 个送货点。

### Q3：如何理解"强制从depot开始"？
**A**：
- **false（默认）**：车辆可以从任意取货点开始，路径更灵活
- **true**：车辆必须先访问depot（索引0），然后开始取送货

### Q4：如何判断配对约束是否满足？
**A**：在可视化中：
- 每对取送货使用相同颜色
- 虚线连接同一对的取货点和送货点
- 检查路线顺序：取货点编号应先于对应送货点

### Q5：训练结果如何应用？
**A**：
1. 使用训练好的模型进行推理
2. 输入实际的取送货需求（地点坐标）
3. 模型输出优化的访问路线
4. 导出为实际调度计划

### Q6：PDP 能否处理多车辆？
**A**：当前实现是单车辆PDP。如果需要多车辆，建议：
- 使用多次单车辆PDP求解
- 或者转为 VRP with Pickup and Delivery (VRPPD) 问题
