## 【第二部分：训练配置 - 1:30-5:00】

**这是平台最核心的部分，分为4个配置步骤。**

### **【第1步：问题配置 - 1:45-2:30】**

**[镜头：第一个卡片]**

首先，我们需要选择**问题类型**。这个下拉菜单包含两类问题：

**路由问题（Routing）：**
- TSP - 旅行商问题（最经典的）
- CVRP - 车辆路径问题（带容量约束）
- VRPTW - 带时间窗的VRP（更复杂，更现实）
- mTSP - 多旅行商问题（多个代理协同）
- 还有 PDP、OP、PCTSP 等

**调度问题（Scheduling）：**
- FFSP - 柔性流水车间调度问题

**[镜头：滚动到数量输入框]**

接下来设置**问题规模**——也就是城市数量或工作数量。这里默认是50个城市。规模越大，模型训练越困难。

**[镜头：条件参数展示]**

这里的巧妙设计是：**不同问题会动态显示不同的参数**。

比如，如果选择 CVRP（车辆路径问题），会自动出现"车辆容量"这个参数。如果选择 VRPTW，会弹出一整块的**时间窗约束**参数——包括时间窗宽度、服务时长、最大配送时间等。

这体现了前端的**动态表单设计**：通过 JavaScript 监听 problem 选择框的变化，动态显示/隐藏对应的参数。

### **【第2步：模型配置 - 2:30-3:30】**

**[镜头：第二个卡片]**

接下来选择**策略模型** —— 这是神经网络的架构。

几个主要模型：
- **Attention Model** - 基于 Transformer 的经典模型，速度快
- **POMO** - 多起点训练，质量更高但更慢
- **SymNCO** - 利用对称性约束，最新最强（新增）
- **HAM** - 异构注意力，专为 PDP 设计
- **MatNet** - 矩阵注意力，专为调度问题设计

**关键设计：** 当你选择不同模型时，下面会自动出现该模型**特有的超参数**。

**[镜头：滚动展示不同模型参数]**

比如选择 **POMO**，会出现"POMO起点数量"输入框；选择 **SymNCO**，会出现"对称增强数量"和"损失权重"等参数。

这又是一个很聪明的设计——避免参数页面过于复杂，只展示当前选择相关的参数。

**再下来是训练算法：**
- **REINFORCE** - 最基础的策略梯度算法
- **PPO** - 业界标准，更稳定
- **A2C** - 平衡的选择

**还有两个推荐按钮：**
- "✨ 使用推荐配置" - 自动填入经过验证的参数组合
- "⚡ 快速原型" - 用最小规模参数，快速测试流程

### **【第3步：训练参数 - 3:30-4:00】**

**[镜头：第三个卡片]**

配置训练的**超参数**：

- **训练轮数（Epochs）** - 默认10，小规模问题建议5-10，大规模建议20-50
- **批次大小（Batch Size）** - 默认512，这会影响显存占用
- **学习率（Learning Rate）** - 默认0.0001，这是优化器的步长

这些参数后面都有中文说明和推荐值，很方便。

**[镜头：GPU卡片]**

### **【训练流程 - 5:15-6:30】**

**[镜头：按钮被禁用，文字变为"⏳ 训练中..."]**

后端做了什么：

1. **接收请求** - Flask 服务器在 `/api/start_training` 接收配置
2. **创建会话** - 为这次训练分配唯一的 `session_id`
3. **启动后台线程** - 不阻塞主线程，避免网页卡死
4. **调用 RL4CO** - 真实的强化学习训练开始

**[镜头：进度显示区域出现]**

```
进度条：显示完成百分比
当前Epoch：显示 5/10
当前Loss：显示损失值（递减）
当前Reward：显示奖励值（递增）
```

这些数据是**实时推送**到前端的。

**技术亮点：使用 Server-Sent Events (SSE)**

```javascript
// 前端代码示例
eventSource = new EventSource(`/api/training_progress/${sessionId}`);
eventSource.onmessage = function(event) {
    const data = JSON.parse(event.data);
    updateProgress(data);  // 更新UI
};
```

SSE 是一个**单向推送**机制：后端可以持续向浏览器发送数据，前端不需要轮询，这样更高效、延迟更低。

## 【第四部分：核心技术架构 - 7:30-9:30】

现在让我讲解这个平台底层的技术实现。

### **【后端架构 - 7:45-8:30】**

**[镜头：打开 VSCode，展示 app.py]**

```
app.py (主入口)
├── 数据库连接管理 (get_db, get_user_manager 等)
├── 缓存层 (SimpleCache 类)
└── 全局变量 (training_status, training_queues)
```

**关键设计 1：请求上下文管理**

```python
def get_db():
    if 'db' not in g:  # Flask 的 g 对象
        g.db = mysql_connector.connect(...)
    return g.db
```

Flask 的 `g` 对象是**请求级别的全局变量**——每个请求都有独立的 `g` 对象，请求结束时自动清理。这样做的好处是**线程安全**。

**关键设计 2：Blueprint 模块化**

```python
from app_auth import auth_bp       # 认证
from app_training import training_bp  # 训练
from app_stats import stats_bp      # 统计
from app_files import files_bp      # 文件管理

app.register_blueprint(auth_bp)
app.register_blueprint(training_bp)
...
```

将原来 1600+ 行的单个 `app.py` 拆分为 6 个 Blueprint 模块，每个模块职责单一。这是**关键的重构改进**。

**关键设计 3：SSE 实时推送**

```python
# app_training.py 中
@training_bp.route('/api/training_progress/<session_id>')
def training_progress(session_id):
    def generate():
        while True:
            if session_id in training_queues:
                message = training_queues[session_id].get(timeout=1)
                yield f'data: {message}\n\n'  # SSE 格式
    return Response(generate(), mimetype='text/event-stream')
```

后端启动训练线程，不断往队列里放数据；SSE 端点不断从队列读数据，推送给前端。这是一个**经典的生产者-消费者模式**。

### **【模型训练核心 - 8:30-9:15】**

**[镜头：modules 文件夹]**

```
modules/
├── rl_training/
│   ├── tsp_trainer.py       # TSP 问题的训练器
│   ├── cvrp_trainer.py      # CVRP 问题的训练器
│   ├── visualizations/      # 可视化模块
│   │   ├── tsp_viz.py
│   │   ├── cvrp_viz.py
│   │   └── common.py        # 公共绘图函数
│   ├── base_trainer.py      # 所有训练器的基类
│   └── training_functions.py # 总入口
├── policies/
│   ├── attention_model_policy.py
│   ├── pomo_policy.py
│   ├── matnet_policy.py
│   └── ...
└── problems/
    ├── tsp.py
    ├── cvrp.py
    └── ...
```

**所以结构是这样的：**

1. `training_functions.py` 根据用户选择的问题类型（problem），调用对应的 Trainer（比如 `tsp_trainer.py`）

2. Trainer 会：
   - 创建 **Environment**（问题环境）- 定义约束、奖励计算
   - 创建 **Policy**（策略模型）- 神经网络
   - 创建 **Algorithm**（训练算法）- REINFORCE/PPO/A2C
   - 用 **PyTorch Lightning** 启动训练循环

3. 训练完成后，调用对应的 Visualizer（比如 `tsp_viz.py`），生成路径图和 GIF
