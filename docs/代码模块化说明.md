# 代码模块化说明

> 项目核心功能模块化改造完成记录

---

## 📋 概述

为提高代码可维护性和复用性，项目进行了模块化改造，将核心训练逻辑、文件管理等功能从 `app.py` 中独立出来。

---

## 🔧 已完成的模块化改造

### 1. RL训练模块（rl_training）

**位置**: `modules/rl_training/`

**迁移内容**:
- `create_route_animation()` - TSP路线动画生成
- `ProgressCallback` 类 - 训练进度回调
- `real_rl4co_training()` - RL4CO训练主函数

**改进效果**:
- `app.py` 减少约 587 行代码
- 训练逻辑独立可测试
- 便于复用和扩展

**使用方式**:
```python
from modules.rl_training import real_rl4co_training

# 在训练线程中调用
real_rl4co_training(config, session_id, user_id, queue, training_status, get_background_db)
```

### 2. 文件管理模块

**改造内容**: 按训练会话分组显示

**改造前**:
- 平铺展示所有文件
- 难以识别哪些文件属于同一次训练

**改造后**:
- 按训练会话分组
- 每个会话显示：对比图、动态GIF、训练曲线、检查点
- 一键删除整个训练会话

**改进效果**:
| 指标 | 改造前 | 改造后 | 提升 |
|------|-------|-------|------|
| **文件关联识别** | 需手动识别 | 自动分组 | ⭐⭐⭐⭐⭐ |
| **删除效率** | 逐个删除 | 一键删除 | **提升4倍** |
| **信息展示** | 仅文件名 | 模型/问题/状态/时间 | ⭐⭐⭐⭐⭐ |

---

## 🎯 模块化原则

1. **单一职责**: 每个模块专注一个功能
2. **松耦合**: 模块间通过参数传递，避免全局依赖
3. **高内聚**: 相关功能集中在同一模块
4. **易测试**: 模块可独立测试

## 📊 模型统计

项目包含 **23 种强化学习模型** 的知识库：

### 按分类统计

- **构造方法（自回归）**: 12个 (52%)
  - AM, POMO, SymNCO, MatNet, PtrNet, HAM, PolyNet, MTPOMO, MVMoE, L2D, HGNN, DF
  
- **构造方法（非自回归）**: 3个 (13%)
  - DeepACO, GFACS, GLOP
  
- **强化学习算法**: 3个 (13%)
  - REINFORCE, A2C, PPO
  
- **改进方法**: 3个 (13%)
  - DACT, N2S, NeuOpt
  
- **传导式强化学习**: 2个 (9%)
  - ActiveSearch, EAS

### 推荐模型

**新手推荐**:
- AM (Attention Model) - 最经典，易于理解
- PtrNet - 开创性工作，概念简洁
- REINFORCE - 最基础的RL算法

**追求性能**:
- POMO ⭐ - 性能与速度平衡最好
- SymNCO ⭐⭐ - 当前SOTA之一
- MatNet - 大规模问题优势明显

**生产环境**:
- PPO ⭐ - 工业界首选，训练最稳定
- A2C - 更稳定的替代方案

## 🔍 技术细节

### 数据库连接管理

- **主线程**: 使用 `get_db()` (Flask g对象)
- **后台线程**: 使用 `get_background_db()` (独立连接)
- **连接清理**: finally 块中确保连接关闭

### 消息队列线程安全

- `training_queues[session_id]` - 每个训练会话独立队列
- 训练线程通过 `queue.put()` 推送进度
- SSE端点通过 `queue.get(timeout=1)` 获取消息

### 文件组织

```
static/model_plots/
  └── user_{user_id}/          # 每个用户独立目录
      ├── training_curves_*.png
      ├── comparison_*.png
      └── animation_*.gif

checkpoints/
  └── user_{user_id}/          # 每个用户独立目录
      └── model-name.ckpt
```
