## 🚀 快速开始（3步）

### 第 1 步：选择问题类型

在训练页面的"第1步：问题配置"中：

1. 打开 **问题类型** 下拉菜单
2. 找到 **调度问题 (Scheduling)** 分组
3. 选择 **FFSP - 柔性流水车间调度 🆕**

### 第 2 步：配置 FFSP 参数

系统会自动显示 FFSP 专用参数面板（蓝色主题）：

```
推荐配置（初次测试）:
┌─────────────────────────────┐
│ 阶段数: 2                   │
│ 机器数/阶段: 2              │
│ 作业数: 10                  │
│ 最小加工时间: 2             │
│ 最大加工时间: 10            │
│ 阶段展平模式: 启用（推荐）  │
└─────────────────────────────┘
```

**实时规模显示**: `2阶段 × 2机器 × 10作业 = 总机器4台，总操作20个`

### 第 3 步：选择训练配置

系统会**自动切换策略为 MatNet**（FFSP必须使用MatNet）

**推荐配置**:
- 策略模型: **MatNet**（自动选择，不可更改）
- 训练算法: **PPO**（推荐）或 A2C（快速）
- 训练轮次: **10-20**（小规模测试）
- Batch Size: **256**
- Learning Rate: **1e-4**

点击 **"开始训练"** ▶️

## ⚙️ 参数说明

### 阶段数 (num_stage)
- **含义**: 生产流程的工序数
- **范围**: 2-10
- **示例**: 
  - 2阶段: 切割 → 组装
  - 3阶段: 切割 → 加工 → 组装
  - 5阶段: 原料 → 预处理 → 加工 → 检验 → 包装

### 机器数/阶段 (num_machine)
- **含义**: 每个阶段的并行机器数
- **范围**: 1-10
- **说明**: 数量越多，并行能力越强，调度越灵活

### 作业数 (num_job)
- **含义**: 需要调度的工件/订单数量
- **范围**: 5-100
- **建议**: 
  - 测试: 10-20
  - 生产: 50-100

### 加工时间范围
- **min_time**: 最小加工时间（1-10）
- **max_time**: 最大加工时间（2-50）
- **说明**: 实际加工时间在此范围内随机生成

### 阶段展平模式 (flatten_stages)
- **启用**: 每个阶段的机器独立编码（推荐）
- **禁用**: 不同阶段的同位置机器共享embedding
- **建议**: 保持默认（启用）

## 🎯 训练目标

### 什么是 Makespan？
**Makespan（完工时间）**: 所有作业完成所需的总时间

```
示例：
作业1: 机器1(t=0-5) → 机器2(t=5-10)
作业2: 机器1(t=5-8) → 机器2(t=10-15)
作业3: 机器1(t=8-12) → 机器2(t=15-18)

Makespan = 18（最后一个作业完成的时间）
```

### 训练目标
- ✅ **最小化 Makespan**（越小越好）
- ✅ 提高机器利用率
- ✅ 减少空闲时间
- ✅ 优化作业顺序

## ❓ 常见问题

### Q1: 为什么必须使用 MatNet？
**A**: FFSP是调度问题，需要处理作业-机器的二维关系。MatNet的**矩阵注意力机制**专为此设计，而Attention Model和POMO是为路由问题优化的。

### Q2: 训练很慢怎么办？
**A**: 
1. 减小问题规模（作业数 = 10）
2. 减少训练轮次（10 epochs）
3. 使用GPU加速
4. 选择A2C算法（比PPO快）

### Q3: 如何提高训练质量？
**A**:
1. 增加训练轮次（50 epochs）
2. 使用PPO算法
3. 增大batch size（512-1024）
4. 适当调整学习率（1e-4 → 5e-5）

### Q4: 甘特图怎么看？
**A**:
- **横轴**: 时间进度
- **纵轴**: 机器编号（M0, M1, ...）
- **彩色块**: 不同的作业
- **块的宽度**: 加工时间
- **块内文字**: 作业编号（J0, J1, ...）

### Q5: flatten_stages 如何选择？
**A**: 
- **大多数情况**: 启用（True）- 性能更好
- **特殊需求**: 禁用（False）- 使用多阶段策略

## 🔧 故障排除

### 错误：RL4CO 库未安装 MatNet
```bash
pip install rl4co --upgrade
```

### 错误：GPU 内存不足
**解决方案**:
1. 减小 batch_size（512 → 256）
2. 减小作业数（50 → 20）
3. 使用CPU训练（较慢）

### 警告：REINFORCE 收敛慢
**解决方案**:
- 切换到 PPO 或 A2C 算法
- FFSP是复杂问题，REINFORCE不够稳定
